ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ТЗ) ПРОЕКТА "EXAMMASTER"

1. ОБЩИЕ СВЕДЕНИЯ
Название проекта: ExamMaster.
Цель: Создание платформы для управления учебными билетами и автоматизированного тестирования знаний через веб-интерфейс и голосового помощника Алису.

2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

2.1. Управление пользователями
- Регистрация: Создание аккаунта с уникальным логином и паролем.
- Аутентификация: Доступ к функционалу только после входа в систему.
- Безопасность: Хеширование паролей с помощью werkzeug.security.
- Удаление данных: Возможность полного удаления аккаунта и связанных с ним данных.

2.2. Управление контентом (Веб-панель)
- Создание предметов: Возможность организации билетов по дисциплинам.
- Импорт билетов: Загрузка вопросов и ответов из .txt файлов (формат: Вопрос;Ответ).
- Управление: Удаление предметов вместе с привязанными к ним билетами.

2.3. Голосовой навык (Интеграция с Алисой)
- Выбор предмета: Нечеткий поиск по названию (коэффициент сходства 0.6).
- Процесс тестирования: Случайная выборка билетов из базы данных.
- Проверка ответов: Сравнение ответа пользователя с эталоном (порог сходства 0.65).
- Навигация: Поддержка команд смены предмета ("назад", "стоп", "другой предмет") и пропуска вопроса ("дальше", "не знаю").

3. ТЕХНИЧЕСКИЙ СТЕК
- Язык: Python 3.x.
- Фреймворк: Flask.
- База данных: SQLite + SQLAlchemy (ORM).
- Фронтенд: HTML5, Bootstrap 5, Jinja2.
- Интеграция: Yandex Alice Skill API.

4. СТРУКТУРА БАЗЫ ДАННЫХ
- Таблица User: id, username, password_hash.
- Таблица Subject: id, title, user_id (внешний ключ).
- Таблица Ticket: id, question, answer, subject_id (внешний ключ).
- Связи: Каскадное удаление (delete-orphan) для всех дочерних элементов.

5. ОСОБЕННОСТИ РЕАЛИЗАЦИИ
- Использование difflib для обработки естественного ввода пользователя.
- Управление контекстом диалога через session_state в API Алисы.
- Применение декораторов Flask-Login для защиты маршрутов.
